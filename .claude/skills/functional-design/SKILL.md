# 機能設計書作成スキル

このスキルは、高品質な機能設計書を作成するための詳細ガイドです。

## 前提条件

機能設計書作成を開始する前に、以下を確認してください:

### 推奨ドキュメント（存在する場合のみ参照）

**ファイルパス**: `docs/core/product-requirements.md`

`gen-all-docs` の規模別生成方針により、PRD は **新規プロジェクトかつ中規模以上**でのみ生成されます。
未生成の場合（既存プロジェクトモード、または小規模プロジェクト）は、以下のフォールバックを使用してください:

| 状況 | フォールバック |
|------|---------------|
| 既存プロジェクトモード（PRD なし） | ソースコード（`src/`）の実装から機能仕様を逆算 |
| 小規模プロジェクト（PRD なし） | `docs/ideas/*.md`（新規時）、`README.md`、`CLAUDE.md` から要件を把握 |

機能設計書は、把握した要件を技術的に実現する方法を詳細化します。

## 既存ドキュメントの優先順位

**重要**: `docs/core/functional-design.md` に既存の機能設計書がある場合、
以下の優先順位に従ってください:

1. **既存の機能設計書 (`docs/core/functional-design.md`)** - 最優先
   - プロジェクト固有の設計が記載されている
   - このスキルのガイドより優先する

2. **このスキルのガイド** - 参考資料
   - 汎用的なテンプレートと例
   - 既存設計書がない場合、または補足として使用

**新規作成時**: このスキルのテンプレートとガイドを参照
**更新時**: 既存設計書の構造と内容を維持しながら更新

## 出力先

作成した機能設計書は以下に保存してください:

```
docs/core/functional-design.md
```
